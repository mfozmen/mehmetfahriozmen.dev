# Illustration Guide — mehmetfahriozmen.dev

## Style

- Modern flat editorial illustration
- Single image per post, no panels
- Colors: deep navy (#0a0f1e), amber (#BA7517), off-white ONLY
- Subtle grain texture
- Clean bold shapes, minimal detail

## Humor

- Deadpan black humor
- One clear joke, readable in a single glance
- Rewards closer inspection with details
- Disaster presented as completely normal

## Character

- Same astronaut in every illustration
- Face never visible (always from behind or profile)
- Setting is wherever the joke lives — spacecraft, labs, museums,
  offices, briefing rooms, warehouses, anywhere. Not restricted to
  space or shuttle environments.

## Reference Image

- File: public/images/writing/hardest-refactor-cover.webp
- Scene: astronaut on video call while cockpit burns behind them

## Text in Illustrations

Diegetic text is allowed — text that exists naturally in the scene's
world. Museum plaques, exhibit labels, signs, placards, equipment
markings, posters on walls, document headers seen in frame, warning
stickers, name tags. If the text would exist in a photograph of this
scene, it can exist in the illustration. Keep it short and legible.

Not allowed:

- Speech bubbles or thought bubbles
- Captions, titles, or headings overlaid on top of the illustration
  (text added to the image rather than living inside the scene)
- Text whose only purpose is to address the viewer or explain the joke

The test: is the text in the scene, or on the scene? In the scene is
fine. On the scene is not.

## ChatGPT Workflow

Illustrations are generated in a dedicated ChatGPT project where the
visual style is already established (flat editorial, navy/amber/off-white
palette, grain texture, astronaut character). No reference image needs
to be attached per request.

Write the prompt as a direct scene description. Do not restate the
style — the project already knows it. Start with the scene.

Rules when writing the scene description:

- Single scene, one clear joke
- Astronaut face never visible
- Keep it simple — complexity kills the humor
- Diegetic text allowed (see "Text in Illustrations" above),
  but keep it short and legible

If a new illustration ever drifts from the established style, regenerate
or start a fresh thread in the project rather than re-explaining the
style in the prompt.
